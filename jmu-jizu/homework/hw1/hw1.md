# 1.什么是存储程序工作方式？

存储程序工作方式是一种计算机工作方式，其核心思想是将程序的指令和数据一起存储在计算机的内存中，并让计算机按照指令的顺序自动执行。这一概念由约翰·冯·诺依曼提出，成为现代计算机的基础结构，因此也被称为“冯·诺依曼结构”。

- 关键特点

    - 程序存储在内存中：程序的指令和数据都存在于计算机的内存中，可以随时被 CPU 读取和执行。
    - 顺序执行：CPU 从内存中按顺序读取指令，一条一条地执行。
    - 可编程性：因为程序和数据在内存中，可以很容易地修改代码，实现计算机的灵活控制。

- 优势

    - 高灵活性：可以运行不同的程序，只需要把新程序载入内存中即可。
    - 自动化：程序自动按照指令顺序运行，不需要人工干预。
    - 通用性：同一台计算机可以执行多种任务，具有通用计算功能。

这种工作方式是现代计算机的基础，也是使计算机可以广泛应用于各个领域的关键原因。

# 4.对源程序的处理有哪两种基本方式？


- 编译（Compilation）：
    在这种方式中，源代码通过编译器转换为目标代码（通常是机器语言或中间语言），然后生成可执行文件。编译过程通常包括词法分析、语法分析、语义分析、优化等阶段。
    - 优点：编译后的程序运行速度快，因为它已被转换为机器语言，直接在计算机上执行。
    - 缺点：编译过程通常需要较长时间，并且在修改源代码后需要重新编译。

- 解释（Interpretation）：
    在这种方式中，源代码由解释器逐行读取并执行。解释器在运行时将源代码翻译为机器指令，并立即执行。
    - 优点：修改源代码后可以立即看到效果，调试相对简单。
    - 缺点：运行速度较慢，因为每次执行时都需要翻译源代码。
